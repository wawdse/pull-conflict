## 충돌사례

### CASE#1
원격레포와 로컬 레포의 커밋 이력이 같은 상황에서 로컬에서
working directory에서의 수정 이후 git pull한다면?

```bash
# pull = fetch + merge
# fetch는 성공인데 merge가 실패한 경우
git pull origin main
```
충돌이 아니므로 그냥 git push하면 된다.

### CASE#2
원격레포에서 commit 이력을 추가했을 때 충돌이 없다.
git pull origin main을 통해 commit이력을 내려 받은 후
병합의 일종으로 볼 수 있으나, 변경사항에 대한 head만
이동하므로 fast forward에 해당함.

### CASE#3
원격에 커밋이력 추가 발생시킴.
또 로컬에서도 별도의 커밋이력을 발생시킴.
단 같은 파일에 대한 수정이 아닐 때 -> 충돌 발생 안함.
git pull시 문제 없이 실행되고 단 fast forward가 아닌
머지 커밋이 일어난다.
이 경우 다시 push가 필요하다.
pull = fetch + merge
### CASE#4 - 가장 빈번한 경우
원격에 commit이력 발생. 동시에 로컬에 별도 커밋이 발생함.
문제는 같은 파일을 커밋한 경우이므로 push할 때 충돌 발생함.
해결방법
git pull origin main 하면 각각 커밋아이디가 있는 경우이므로
(참고로 case#5번은 한쪽만 커밋이 일어난 경우)
vscode GUI로 이미지 화면을 제공해줌.
틀린 부분에 대해서 병합파일 생성
병합파일 적절하게 수정했다면
git add .
git commit -m "메시지"
git push origin main
